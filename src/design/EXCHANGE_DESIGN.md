# Xyra Labs Exchange - Design Document

## 1. Executive Summary
This document outlines the design and architecture for the **Xyra Labs** Next.js exchange application. The layout is strictly based on the provided reference visual, prioritizing a professional, high-density trading interface with a responsive and aesthetic design.

## 2. Layout Structure
The application uses a **Holy Grail Layout** optimized for trading data density.
- **Viewport**: 100% width/height, fixed (non-scrolling window, internal scrolling areas).
- **Grid System**: CSS Grid.

### High-Level Grid Areas
1.  **Header** (`he`): Top navigation and user info (h ~60px).
2.  **Ticker Bar** (`ti`): Pair stats and market info (h ~50px).
3.  **Main Content** (`ma`): 3-column/panel layout.
    -   **Left**: Charting Area (Flexible width, ~60-70%).
    -   **Right** (Split vertically):
        -   **Orderbook** (Top half).
        -   **Trade Form** (Bottom half).
4.  **Bottom Panel** (`bo`): Collapsible/Tabbed User Activity (Positions, Orders).

## 3. Component Tree & Responsibilities

### 3.1. App Shell
-   **`Layout.js`**: Main wrapper. Key responsibility: manage global theme and grid structure.
-   **`Header.js`**:
    -   **Logo**: Xyra Labs (Text/Icon).
    -   **NavMenu**: Links (Perps, Swap, Leaderboard, etc.).
    -   **UserControls**: VIP Btn, "Buy USDT", Deposit, Sign In, Notifications, Settings.

### 3.2. Market Data Region (Ticker)
-   **`TickerBar.js`**:
    -   **`PairSelector`**: Dropdown for "BTC-PERP".
    -   **`MarketStats`**: Stateless display of:
        -   Mark Price
        -   24h Change (Color-coded)
        -   Oracle Price
        -   24h Volume
        -   Funding Rate / Countdown.

### 3.3. Main Trading Workspace
-   **`ChartSection.js`**:
    -   Container for the chart library (or mock).
    -   **`ChartToolbar.js`**:
        -   Time intervals (1m, 5m, 1h...).
        -   Tools (Indicators, Display settings).
    -   **`TVChartContainer.js`**: (Placeholder for TradingView).
-   **`OrderBook.js`**:
    -   **Header**: Tabs for "Orderbook" / "Recent Trades".
    -   **TableHeader**: Price(USDT), Size(Asset), Sum(Asset).
    -   **`AsksList`**: Red rows (Sell orders).
    -   **`TickerSpread`**: Middle bar showing current price & spread.
    -   **`BidsList`**: Green rows (Buy orders).
-   **`TradeInterface.js`** (Right Panel):
    -   **`AccountInfo`**: "Profile 1 / 0.00 USDT".
    -   **`MarginMode`**: Tabs (Isolated / Hedge).
    -   **`OrderType`**: Tabs (Open/Close, Market/Limit).
    -   **`LeverageControl`**: Slider and Input (e.g., 20x).
    -   **`OrderInputs`**: Amount field, Percentage slider (0-100%).
    -   **`TPSLInput`**: Checkbox and hidden fields for TP/SL.
    -   **`SubmitButton`**: Large "Connect Wallet" or "Buy/Sell" button.
    -   **`OrderSummary`**: Details (Liq Price, Margin, Fees).

### 3.4. User Dashboard (Bottom)
-   **`PositionsPanel.js`**:
    -   **Tabs**: Position, Open Orders, Order History, Trade History, etc.
    -   **Table**: Data grid for active positions.
    -   **EmptyState**: "Sign in to see your Position".

## 4. Technical Stack
-   **Framework**: Next.js 16 (Pages Router preferred for this structure or App Router if configured).
-   **Styling**: Tailwind CSS v4.
    -   *Strict adherence to dark mode aesthetics (Slate/Gray palette, Emerald for Buys, Rose for Sells).*
-   **Icons**: Lucide React or Heroicons.
